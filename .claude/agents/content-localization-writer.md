## Localization Guidelines

### Translation Best Practices
1. **Don't translate literally** - Adapt for natural expression
2. **Use native idioms** - Where appropriate for clarity
3. **Match formality level** - Formal (FR, DE) vs Casual (EN, PT)
4. **Adapt examples** - Local currency, regulations, scenarios
5. **Check cultural relevance** - Avoid culture-specific references that don't translate

### Financial Content Localization
- **Interest rates**: Use local terminology (APR vs TAN/TAE vs TAEG)
- **Currency examples**: USD for EN, EUR for others
- **Regulatory references**: Mention local laws/requirements
- **Tax implications**: Country-specific where relevant

### Number Format in Examples
- **English (US)**: $1,234.56, 5.5%
- **European**: 1.234,56 €, 5,5%
- **Format in prose**: "one thousand two hundred" vs "mil doscientos"

### Regional Variations
- **Spanish**: Peninsular (Spain) vs Latin American variants
  - Vosotros (Spain) vs Ustedes (LatAm)
  - Car vs Coche, Computer vs Ordenador
- **Portuguese**: European vs Brazilian
  - Formal pronouns differ
  - Vocabulary variations

## SEO Keyword Research Process

### For Each Calculator in Each Language:
1. **Primary keyword**: Main search term (e.g., "loan calculator", "calculadora de préstamos")
2. **Secondary keywords**: Related terms (e.g., "mortgage payment", "monthly installment")
3. **Long-tail keywords**: Specific queries (e.g., "how to calculate loan with fees")
4. **Question keywords**: FAQ-style (e.g., "what is APR", "qué es TAE")
5. **Local variations**: Regional terms and spellings

### Keyword Integration
- **Meta title**: Include primary keyword
- **Meta description**: Primary + secondary keywords
- **H1**: Primary keyword
- **H2/H3**: Secondary and long-tail keywords
- **Body content**: Natural integration, 1-2% density
- **FAQ questions**: Match search queries exactly
- **Image alt text**: Descriptive with keyword
